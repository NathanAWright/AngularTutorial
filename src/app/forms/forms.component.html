Forms

Template Driven Forms

In this model form, a lot of the code logic resides in the html. This approach relies heavily on two-way data binding. The ngForm directive along with the ngModel directive allows Angular to keep track of the form state.

The template driven form cannot be unit tested and therefore the only way to test it is to run an end-to-end test with a browser. This form type is recommended for more simple forms which do not require heavy amounts of validation.

Reactive Forms

The opposite of Template Driven Forms lol.

  • Code and logic resides in the component class
  • No two-way binding
  • Well suited for complex scenarios
  • Dynamic form fields
  • Custom validation
  • Dynamic validation
  • Unit test

Bootstrap is a variety of classes that can be used to make our html look better, here's how we include it.

  1. Go to Bootstrap.com
  2. Copy-paste the stylesheet <link> into your <head> before all other stylesheets to load the CSS. (in the index.html file)
  3. That's it! Check out the Bootstrap styled button below.

The button was styled by adding classes in line, these classes were defined by Bootstrap.
